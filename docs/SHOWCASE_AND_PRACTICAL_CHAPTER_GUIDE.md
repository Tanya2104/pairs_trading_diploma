# Гайд: как показывать проект научруку и писать практическую главу

Этот документ — «каркас» для защиты промежуточных результатов и для практической части диплома по проекту парного трейдинга.

## 1) Что уже реализовано в проекте (для короткого устного отчёта)

Используй формулу **«данные → метод → стратегия → бэктест → сравнение методов → вывод»**:

1. **Данные (MOEX)**
   - загрузка цен по списку тикеров и диапазону дат;
   - очистка, контроль качества, синхронизация временных рядов.
2. **Эконометрический метод**
   - поиск коинтегрированных пар;
   - оценка p-value, beta, R² и half-life.
3. **Торговая логика**
   - сигналы по Z-оценке спреда (entry/exit);
   - ограничение по максимальному времени удержания позиции.
4. **Бэктест**
   - расчёт доходности рыночно-нейтрального портфеля пары;
   - метрики: доходность, Sharpe, просадка, win-rate, число сделок.
5. **Сравнение подходов**
   - коинтеграционный метод сравнивается с корреляционным бенчмарком;
   - победитель выбирается по приоритету: Sharpe → Total Return → Max Drawdown.

## 2) Чек-лист демонстрации научруку (10–15 минут)

### Шаг 1. Подготовить окружение

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Шаг 2. Быстрая проверка пайплайна

```bash
python core/smoke_pipeline.py
```

Ожидаемый результат: `SMOKE_TEST_OK`.

### Шаг 3. Запуск интерфейса

```bash
streamlit run app/streamlit_app.py
```

### Шаг 4. Что показать на экране

1. Выбор тикеров и периода.
2. Блок качества данных (число дней/тикеров/пропусков).
3. Найденная лучшая пара (p-value, beta, R², half-life).
4. Графики:
   - спред + Z-score;
   - кривая капитала.
5. Метрики бэктеста и интерпретация устойчивости.
6. Сравнение коинтеграции и корреляции.

## 3) Структура практической главы (готовый скелет)

Ниже можно брать как шаблон оглавления практической главы.

## 3.1 Постановка практической задачи

- Цель: разработать и проверить рыночно-нейтральную стратегию парного трейдинга на данных MOEX.
- Объект: временные ряды цен ликвидных акций.
- Предмет: статистические зависимости (коинтеграция) и их использование для генерации торговых сигналов.

## 3.2 Описание данных и предобработки

- Источник данных (API/кэш), временной интервал, список тикеров.
- Метрики качества данных:
  - число наблюдений,
  - доля пропусков,
  - исключение рядов с избытком пропусков,
  - синхронизация дат.

**Что вставить в текст:**
- таблицу параметров выборки;
- скрин блока «Качество данных» из Streamlit.

## 3.3 Метод формирования пар

- Критерий отбора: тест коинтеграции при пороге p-value.
- Параметры регрессии спреда: beta, R².
- Интерпретация half-life как прокси скорости возврата к среднему.

## 3.4 Реализация стратегии

- Нормализация спреда через rolling Z-score.
- Правила входа/выхода:
  - вход при |Z| > entry;
  - выход при |Z| < exit;
  - принудительный выход по max_holding_days.

## 3.5 Бэктест и метрики

Рекомендуемый набор метрик:
- Совокупная доходность (Total Return)
- Годовая доходность
- Sharpe Ratio
- Максимальная просадка
- Win-rate по дням
- Количество сделок

## 3.6 Сравнение с альтернативой

- Бенчмарк: выбор пары по максимальной |корреляции|.
- Единые параметры для fair comparison.
- Вывод о том, какой метод устойчивее на выбранной выборке.

## 3.7 Ограничения и направления улучшения

- отсутствие комиссий/проскальзывания;
- риск переобучения на одном историческом окне;
- необходимость walk-forward и out-of-sample проверки;
- необходимость стресс-тестов по режимам рынка.
